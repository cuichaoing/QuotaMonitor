## 二、v1.0 用户能做什么

打开应用后，状态栏出现一个柱状图图标。

**首次启动** → Popup 顶部有黄色引导条 → 点"打开设置" → 填 API Key → 保存 → 自动开始抓取 → 状态栏出现 `K 35 M 60 G 12` 实时数字。

**日常使用**：
- 点击状态栏图标 → 看 3 平台 5h 配额 + 重置倒计时
- 系统通知在 75/90/99% 时弹出
- 状态栏颜色按最高百分比变化（绿/黄/橙/红）
- 设置面板可调阈值、暂停平台、配置 Bark

**退出后** → API Key 还在 Keychain → 下次启动无需重新配置。

## 三、关键能力清单

### 3.1 数据抓取
- 3 平台并行（actor + withTaskGroup）
- 平均 200ms 完成一轮
- 数据平滑：同窗口 used 不倒退
- Key 缺失引导：未配置 key 自动跳过

### 3.2 智能节流
- 4 级：活跃 30s / 普通 60s / 空闲 5m / 休眠暂停
- NSWorkspace 监听 IDE 前台
- 阈值可由用户在高级设置调整

### 3.3 安全
- API Key 走 macOS Keychain（`kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly`）
- 任何敏感信息不出现在日志中
- 配置文件脱敏 0 命中

### 3.4 告警
- 系统通知 3 级（75/90/99）
- 5h 窗口指纹去重（升级才发，降级不发）
- Bark 远程推送（可选）
- 危险级别带 alarm 音

### 3.5 UI
- 状态栏紧凑显示
- Popup 360x480 紧凑面板
- 设置 520x700 弹窗
- 首次启动引导
- 颜色语义化（normal/warning/critical/error）

## 四、单元测试覆盖

**95 个测试用例，0 失败，~0.5s 完成**

| 测试文件 | 用例 | 核心断言 |
|----------|------|----------|
| ProviderKind | 7 | 3 平台枚举字段 |
| QuotaWindow | 6 | 百分比/remaining |
| QuotaError | 11 | isRetryable / isUserActionable |
| KeychainStore | 6 | CRUD |
| ProviderQuota | 3 | 5h 指纹边界 |
| KimiProvider | 8 | User-Agent / Bearer / 401/429/500 |
| MiniMaxProvider | 4 | "current_interval_usage_count 是剩余" |
| GLMProvider | 5 | 裸 Token / 5H 窗口优先 |
| NetworkingService | 5 | 并行 < 0.5s / 平滑 / Key 缺失 |
| SmartRefreshScheduler | 6 | 4 级间隔 / sleeping 跳过 |
| SettingsStore | 18 | Keychain + UserDefaults 持久化 |
| AlertStateMachine | 12 | 3 级 + 5h 去重 + 升级/降级 |
| BarkClient | 4 | URL 构造 / 编码 / 4xx |

### 核心反直觉断言（已固化）

- **MiniMax**：`current_interval_usage_count` 是"剩余"非"已用"，used = total - remaining
- **GLM**：Authorization 头禁止 "Bearer " 前缀，必须裸 Token
- **Kimi**：必须带 `User-Agent: KimiCLI/1.30.0` 才能命中 Coding Plan 专属额度
- **数据平滑**：同 5h 窗口内 used 不倒退
- **告警去重**：同窗口同 provider 升级才发，降级不发

## 五、关键技术决策

| 决策 | 原因 |
|------|------|
| URLProtocol + async handler | 测试可拦截，Task.detached 避免 actor 继承 |
| actor + withTaskGroup | 真正并行而非串行 |
| @MainActor ObservableObject | SwiftUI 兼容 + 线程安全 |
| 5h 窗口指纹 = `floor(epoch/18000)` | 跨平台一致的去重粒度 |
| Keychain + UserDefaults 分层 | 敏感 vs 配置分离 |
| Bark over Cloudflare Worker | 更简单，单文件 200 行 |
| `NetService`/`NSPopover` 混合 | macOS 12+ 兼容性 + 调试便利 |
| 单例 `Settings.shared` | 避免多份配置实例 |

## 六、已知边界与改进方向

### 当前版本
- 仅 macOS 12+（用 NSPopover + SwiftUI 桥接）
- Bark 自建服务器需手动改 URL
- 状态栏 title 在中文 locale 下不会本地化（K/M/G 是固定短码）

### 未来可扩展
- 燃烧速度趋势图（需要历史快照存储）
- 菜单栏点击 "设置" 之外的 "快速切换平台" 子菜单
- 启动时检测 Keychain 授权弹窗（首次保存 Key 时）
- TestFlight 打包 + 签名

## 七、上手清单（给用户）

1. `cd <PROJECT_ROOT>`
2. `swift build -c release` （编译 release 版本）
3. `./.build/release/QuotaMonitor` 启动
4. 状态栏出现图标 → 点击 → 看到首次启动引导
5. 点"打开设置" → 填 3 个平台的 API Key → 保存
6. 状态栏出现 `K XX M XX G XX` 实时数字
7. 设置里勾选 Bark 推送（可选）→ iPhone 接收

详见 `USER_GUIDE.md`。
