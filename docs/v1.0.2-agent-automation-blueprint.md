## 1. 三层自动化能力（L1 / L2 / L3）

### L1：GitHub Actions（你 push 后自己跑，全自动）

**原理**：在 `.github/workflows/*.yml` 写 YAML，GitHub 服务器读 → 跑 macOS runner → 自动执行。

**现状**：
- ✓ `test.yml`：PR/push → 跑 swift test（已存在）
- ✓ `release.yml`：push tag → 跑 build + 产 .dmg + 发 Release（**刚加**）

**未来可加**：
- `pr-test.yml`：PR 提交 → 跑全套 test
- `dependabot.yml`：自动检测依赖更新
- `codeql.yml`：自动代码安全扫描

**能做到**：
| 能力 | 自动化 |
|------|------|
| 自动跑测试 | ✓ 全自动 |
| 自动发 Release（产 dmg） | ✓ 全自动 |
| 自动检查依赖过期 | ✓ 全自动 |
| 自动安全扫描 | ✓ 全自动 |
| 自动 review PR | △ 有限（见下） |

**局限**：GitHub Actions 本身**不能**主动调 AI——它只能跑脚本/命令。

### L2：gh CLI（你触发我，我执行，半自动）

**原理**：你装 `gh` CLI + OAuth 登录 → 我能用 `gh` 命令直接操作你的 GitHub。

**能做到**：
| 能力 | 命令 |
|------|------|
| 创建 repo | `gh repo create QuotaMonitor --public` |
| push 代码 | `git push`（用 gh 提供的认证） |
| 提 PR | `gh pr create --fill` |
| 审查 PR | `gh pr diff 8` + 我写 review + `gh pr review --comment` |
| 合并 PR | `gh pr merge 8 --squash` |
| 关 issue | `gh issue close 5 -c "已修"` |
| 触发 workflow | `gh workflow run release.yml` |
| 发 Release | `gh release create v1.0.2 build/*.dmg` |
| 看 CI 状态 | `gh run watch` |

**典型工作流**（"你触发我跑"）：
```
你：「发布 v1.0.2」
我：
  1. git tag v1.0.2
  2. git push --tags   → 触发 L1 release.yml
  3. gh run watch       → 等 Actions 跑完
  4. gh release create  → （L1 已经做了，verify 即可）
  5. 发推文
```

### L3：Webhook + Claude API（外部 trigger，全自动）

**原理**：GitHub 事件 → 外部 server / GitHub Action → 调 Claude API（不是 Claude Code） → AI 处理。

**能做到**：
| 能力 | 自动化 |
|------|------|
| 自动 review PR（带 AI 分析） | ✓ 全自动 |
| 自动分析 issue 报告原因 | ✓ 全自动 |
| 自动给 PR 写 fix | △ 有限（AI 写代码需要 review） |
| 自动给 issue 评论 | ✓ 全自动 |

**实现路径 A：用 GitHub Actions 调 Claude API**

```yaml
# .github/workflows/ai-review.yml
name: AI PR Review
on:
  pull_request:
    types: [opened, synchronize]
jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - name: Get diff
        run: gh pr diff ${{ github.event.pull_request.number }}
      - name: Call Claude API
        run: |
          # 调 Claude API 传 diff + prompt
          curl -X POST https://api.anthropic.com/v1/messages \
            -H "x-api-key: ${{ secrets.ANTHROPIC_API_KEY }}" \
            -d '...'
      - name: Post review
        uses: actions/github-script@v7
        with:
          script: |
            github.rest.pulls.createReview({
              owner: context.repo.owner,
              repo: context.repo.repo,
              pull_number: context.issue.number,
              body: 'AI Review: ...',
              event: 'COMMENT'
            })
```

**实现路径 B：本地 daemon 监听 webhook**

```python
# scripts/webhook_listener.py （伪代码）
from flask import Flask
import requests

app = Flask(__name__)

@app.route('/webhook', methods=['POST'])
def handle():
    event = request.headers.get('X-GitHub-Event')
    if event == 'pull_request':
        # 调 Claude API
        review = claude_review(request.json)
        # 评论回 PR
        gh.post_review(request.json['pull_request']['number'], review)
    return '', 200

# 跑：python webhook_listener.py（用 launchd 保活）
```

**局限**：
- 调 Claude API **要付费**（按 token）
- 每次 review 大约 $0.10-0.50
- 安全风险：API key 存 GitHub Secrets

### 三层对比

| 维度 | L1 Actions | L2 gh CLI | L3 Webhook + API |
|------|-----------|-----------|-----------------|
| 全自动 | ✓ | ✗ 需我 | ✓ |
| 我参与 | ✗ | ✓ | ✗ |
| 费用 | 免费（2000 min/月） | 免费 | 付费（Claude API） |
| 风险 | 低（YAML 错只影响 build） | 中（你 token 给我） | 中（API key 泄露） |
| 适用 | 标准化流程 | 灵活决策 | 大规模自动化 |
| **推荐** | **先用** | **你授权后** | **需要时再上** |

## 2. 推荐实施路径

### 阶段 1：装 gh + 登录（5 min，立刻）

**这一步解锁 L2。** 之后你触发我，我能跑任何 gh 命令。

```bash
brew install gh

# 选 GitHub.com → HTTPS → Login with a web browser
# 跳到浏览器点 Authorize
```

完成后给我说一声"gh 装好了"，我跑首次 push。

### 阶段 2：首次 push + 跑 CI（10 min）

```bash
cd /Users/cc/Documents/我的代码/2026.6.17-QuotaMonitor
gh repo create QuotaMonitor --public \
  --description "macOS 状态栏原生应用：实时监控 Kimi / MiniMax / GLM 三大 AI 平台 5h 配额" \
  --source=. --push
```

L1 触发：test.yml 自动跑 → 看到 ✓ 105/105 passing

### 阶段 3：写 PR Review 流程（1 h，你授权后）

**日常使用**：
```
你：「看一下 PR #8」
我：
  1. gh pr diff 8
  2. 我看 diff + 跑 swift test
  3. 我写 review（approve / request changes / comment）
  4. gh pr review 8 --comment -b "review 内容"
  5. 可选：gh pr merge 8 --squash --delete-branch
```

### 阶段 4：自动发版流程（30 min）

```bash
# 一次性写个发布脚本
cat > scripts/release.sh <<'EOF'
#!/bin/bash
set -e
VERSION=$1
swift build -c release
swift test
git tag "v$VERSION"
git push --tags   # 触发 L1 release.yml
gh run watch      # 等 Actions 跑完
gh release view "v$VERSION"  # 验证
EOF
chmod +x scripts/release.sh
```

之后：
```
你：「发布 v1.0.3」
我：./scripts/release.sh 1.0.3
```

### 阶段 5：自动修 bug（2 h，长期）

**这是 L3 范畴，需要 Claude API 或 launchd 守护进程。**

最小可行方案（Actions + Claude API）：
1. 写 `.github/workflows/ai-triage.yml`：issue opened → 调 Claude API 分析
2. 调 Claude：issue 内容 + 代码库 → 输出"是否 bug / 修复方向 / 严重程度"
3. Actions 自动加 label（`bug` / `enhancement` / `wontfix`） + 自动评论

完整自动修：
1. issue → L3 AI 分析 → 写 fix 代码 → 提 PR
2. 需要 Claude Code 不是 API（更复杂）

## 3. 关键诚实声明

### 我能做到
- ✓ 用 `gh` 操作你的 GitHub（你授权后）
- ✓ 写 GitHub Actions YAML（已经做了 2 个）
- ✓ 写 webhook 监听脚本

### 我做不到

对于 QuotaMonitor **一个人项目**，**L1 + L2 完全够用**。L3 是"杀手锏"但 ROI 较低。
