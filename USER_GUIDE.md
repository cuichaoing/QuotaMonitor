# QuotaMonitor 用户使用指南

macOS 状态栏原生应用，监控 Kimi / MiniMax / GLM 三大 AI 平台 5h 配额。

---

## 1. 安装

### 方式 A：从源码编译

```bash
cd /path/to/2026.6.17-QuotaMonitor
swift build -c release
./.build/release/QuotaMonitor
```

### 方式 B：用 Xcode

```bash
open Package.swift
# 然后按 ⌘R 运行
```

应用启动后会驻留在 macOS 顶部状态栏（不显示在 Dock、不抢焦点）。

**系统要求：macOS 13 (Ventura) 及以上。**

---

## 2. 首次配置

### 2.1 打开设置

点击状态栏图标 → 弹出 Popup → 左下角"设置"。

或：右键点击状态栏图标 → 设置...

或：⌘ + 逗号（在 App 内时）

### 2.2 填入 API Key

| 平台 | 用途 | Key 获取 |
|------|------|---------|
| Kimi Code | 月之暗面编程套餐 | https://kimi.com → 设置 → API Key |
| MiniMax | MiniMax 编程套餐 | https://platform.minimax.io → Coding Plan |
| 智谱 GLM | 智谱 AI 编程套餐 | https://bigmodel.cn → API Keys |

操作步骤：
1. 在对应平台行填写 Key
2. 点击"保存"
3. 该平台会自动启用（已配置 标记出现）

### 2.3 启用系统通知

首次启动会弹系统对话框：

```
QuotaMonitor 想给您发送通知
[不允许] [允许]
```

点"允许"。之后配额达到 75%/90%/99% 时会原生通知。

### 2.4 启用 Bark（可选）

iPhone 接收推送：

1. iPhone 装 **Bark** App（App Store 搜"Bark"）
2. 打开 Bark，复制顶部的 device key（形如 `abc123def456...`）
3. 回到 QuotaMonitor 设置 → 勾选"Bark 推送"
4. 粘贴 device key → 保存

之后告警会同时推到 iPhone。

## 3. 日常使用

### 3.1 状态栏数字

```
K 35 M 60 G 12
```

- `K` = Kimi，`M` = MiniMax，`G` = GLM
- 后面数字 = 已用百分比
- 颜色 = 最高百分比对应：
  - 绿色 < 75%
  - 黄色 75-90%
  - 橙色 90-99%
  - 红色 ≥ 99%

### 3.2 Popup 面板

点击状态栏图标展开：

```
+----------------------------+
| QuotaMonitor        [⟳]    |   <- 标题 + 手动刷新
+----------------------------+
| Kimi Code          35%     |
| ▓▓▓▓▓░░░░░░░░░░░░░░░░░░░   |
| 5h 窗口  重置于 03:42:15    |
+----------------------------+
| MiniMax Coding     60%     |
| ▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░░░░   |
| 5h 窗口  重置于 01:23:45    |
+----------------------------+
| 智谱 GLM           12%     |
| ▓▓▓░░░░░░░░░░░░░░░░░░░░░   |
| 5h 窗口  重置于 04:55:30    |
+----------------------------+
| 设置  刷新于 14:32:05  退出 |   <- 底部
+----------------------------+
```

### 3.3 告警行为

| 阈值 | 通知内容 | 提示音 |
|------|---------|--------|
| 75% | Kimi 已用 75%，请注意配额消耗 | 默认 |
| 90% | Kimi 已用 90%，接近上限，建议停止消耗 | 默认 |
| 99% | Kimi 已用 99%，配额即将耗尽 | alarm 警报 |

去重规则：
- 同一 5h 窗口内同一等级只发一次
- 等级**升级**会再发（如 75% → 90%）
- 等级**降级**不发（如 90% → 75%）
- 跨 5h 窗口自动重置

### 3.4 智能节流

应用根据你的 Mac 使用状态自动调节抓取频率：

| 状态 | 频率 | 触发条件 |
|------|------|----------|
| 活跃 | 30s | 有 IDE（VSCode/Xcode/Cursor）前台 + 鼠标键盘活跃 |
| 普通 | 60s | 普通前台应用 |
| 空闲 | 5m | 长时间无活动 |
| 休眠 | 暂停 | 系统睡眠 |

默认频率可在设置 → 高级 调整。

## 4. 进阶配置

### 4.1 暂停某平台

设置 → 取消勾选某平台的 checkbox → 该平台不再抓取，状态栏数字也消失。

### 4.2 自定义阈值

设置 → 颜色阈值 → 拖动 slider。

例如想让告警更敏感：
- 黄色警告：50（默认 75）
- 橙色严重：75（默认 90）
- 红色危险：95（默认 99）

### 4.3 自定义 Bark 服务器

如果自建 Bark 服务器：设置 → Bark 推送 → 改"服务器"字段为你的 URL。

---

## 5. 开机自动启动

设置 → 启动 → 打开"登录时自动启动 QuotaMonitor"。

首次开启时，macOS 会要求你在"系统设置 → 登录项"中允许 QuotaMonitor。点击设置面板中的"打开系统设置"按钮，勾选 QuotaMonitor 即可。

---

## 6. 故障排查

### Q: 状态栏只显示 `K -- M -- G --`

A: 还没配置任何 API Key，或全部平台抓取失败。打开设置检查：
- Key 是否填了
- 平台是否启用（勾选）
- 平台卡片是否有红色错误提示

### Q: 某个平台一直 401

A: Key 无效或过期。检查：
- Kimi User-Agent 是否被服务端拒绝（极少）
- GLM Key 不能含 "Bearer " 前缀
- MiniMax Key 必须是 Coding Plan Key，不是普通 Chat Key

### Q: 通知不弹出

A: 检查：
- 系统设置 → 通知 → QuotaMonitor → 允许通知
- 应用启动时是否同意了通知权限

### Q: Bark 推送不到 iPhone

A: 检查：
- iPhone Bark App 是否在线
- device key 是否复制完整
- 自建服务器地址是否正确

---

## 7. 隐私

- 所有 API Key 存 macOS Keychain（硬件级加密）
- 配置（阈值/启用平台）存 UserDefaults（本地）
- 应用不收集任何使用数据
- 网络请求只发到 3 个平台官方 API +（可选）你配置的 Bark 服务器

---

## 8. 退出应用

- 状态栏右键 → 退出
- 或 popup 左下角"退出"
- ⌘Q 在 popup 内时也有效

退出后下次重启会从 Keychain 重新读取 Key，无需重新配置。
